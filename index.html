Trade AI : The Prompt Master GuideAI 프롬프트 활용 가이드

복잡한 관세, 통관, 물류 업무를 스마트하게 해결하기 위한 무역 전문가 전용 AI 프롬프트

1. 프롬프트의 세 가지 유형

무역 실무의 복잡도에 따라 프롬프트의 구조를 선택하세요. 맥락이 중요한 상담 업무부터 데이터 중심의 규제 조회까지 최적의 형식이 존재합니다.

1) 서술형

특징: 맥락 중심, 재사용 낮음, 유연성 높음

업무의 맥락과 배경을 상세히 설명하여 AI가 현재 상황을 완벽히 이해하고 유연하게 대처하도록 유도합니다.

✅ 예시
당신은 10년 경력의 무역 전문가 입니다.
중소기업 수출 담당자가 HS code를 처음 배우는 상황입니다.
이 사람에게 HS code의 개념과 중요성을 설명해 주세요.

2) 지침형

특징: 규칙 중심, 재사용 중간, 유연성 중간

단계별 절차나 명확한 제약 조건을 제시하여 사용자가 의도한 정확한 결과물을 출력하도록 강제합니다.

✅ 예시
1. 함수명은 snake_case로 작성
2. 각 함수의 docstring 포함
3. 예외 처리 반드시 포함
4. 결과는 json 형식으로 반환

3) 함수형 구조

특징: 구조 중심, 재사용 높음, 유연성 낮음

입력값과 출력값을 명확히 정의하여 프로그램처럼 일관된 형식의 결과를 생성합니다. 자동화 시스템 연동에 유리합니다.

✅ 예시
입력 : {product_name} , {hs_code} , {country}
처리 : 해당 국가의 관세율 및 수입 규제 조회
출력 : {
  "관세율" : "",
  "규제사항" : [],
  "필요서류" : []
}

2. 가장 기본적인 제로샷, 원샷, 퓨샷 프롬프팅

AI에게 정답의 '예시'를 몇 개 제공하느냐에 따라 답변의 정확도와 형식이 결정됩니다. 복잡한 무역 실무 데이터일수록 더 많은 예시(퓨샷)가 필요합니다.

1) 제로샷 (Zero-shot)

특징: 예시 없이 바로 지시하는 방식입니다. AI의 사전 학습 데이터에만 의존하여 결과를 도출합니다.

✅ 예시
이 제품 정말 맘에 들어.
강아지에 대해서 설명해주세요.

2) 원샷 (One-shot)

특징: 예시를 1개 제공하는 방식입니다. AI에게 답변의 '형식'이나 '톤'을 인지시킬 때 효과적입니다.

✅ 예시
감정을 분류해 주세요
예시)
"배송이 너무 늦어서 실망했어요"
감정 : 부정

이 제품 정말 맘에 들어요
감정 : (긍정)

3) 퓨샷 (Few-shot)

특징: 예시를 2개 이상 제공하여 복잡한 패턴을 학습시킵니다. 무역 용어 분류나 코드 매칭 시 정확도가 가장 높습니다.

✅ 예시: HS Code를 분류해 주세요
예시)
품목 : "면으로 만든 남성용 티셔츠" -> 6109.10
품목 : "폴리에스테르 소재 여성 블라우스" -> 6206.40
품목 : "아동용 데님 바지" -> 6203.42

품목 : "울 혼방 남성 재킷"
HS Code : (6203.31)

* 가장 권장하는 방식입니다.

3. 페르소나 (Persona)

AI에게 특정한 역할과 인격을 부여하여 답변의 관점과 전문성을 제어하는 기법입니다. 누구의 입장에서 말하느냐에 따라 결과물의 깊이가 달라집니다.

페르소나 패턴

동일한 질문이라도 부여된 페르소나에 따라 분석의 핵심 포인트가 변화합니다.

✅ 예시: "이 계약서 검토해줘"
  • 페르소나 없음 : 일반적인 조항 설명
  • 변호사 : 법적 리스크 및 독소 조항 중심 분석
  • CFO : 재무 조건, 수익성 및 결제 방식 분석
  • 신입사원 : 용어 풀이 중심의 쉽고 친절한 설명

페르소나 구성 요소

좋은 페르소나를 만드는 4가지 필수 공식입니다.

1. 당신은 [역할] 입니다.
2. [경력/배경]을 가지고 있으며,
3. [전문분야]에 특화되어 있습니다.
4. [말투/태도]로 응답하세요.

예시1. 관세사 페르소나

당신은 20년 경력의 관세사 입니다. FTA 협정세율과 원산지 규정의 전문가로 중소기업 무역 담당자에게 복잡한 내용을 쉽고, 실용적으로 설명하는 것이 특기입니다.

예시2. 비평가 페르소나

당신은 까다로운 개발자 입니다. 코드 리뷰 시 성능, 보안, 가독성을 엄격하게 평가하며 칭찬보다 개선점을 먼저 지적하는 스타일입니다.

멀티 페르소나 (Multi-Persona)

한 번에 여러 전문가를 소환하여 다각도에서 솔루션을 검토하고 찬반 의견을 수렴합니다.

✅ 예시: 우리 제품의 동남아 수출 전략 분석
- 김부장 : 영업 15년, 낙관적이며 현장 실무 중심
- 이박사 : 시장조사 전문, 데이터 중심이며 매우 신중함
- 박대표 : 스타트업 창업가, 속도 중시 및 리스크 감수 성향

지시 : 각자의 관점에서 전략에 대한 찬반 의견을 제시하세요.

4. 마크다운을 이용한 읽기 쉽고 명료한 프롬프트 작성법

AI는 텍스트를 구조적으로 파싱(쪼개기)하여 이해합니다. 마크다운을 이용하면 역할, 조건, 출력 형태가 명확히 구분되어 더 정확하고 일관된 응답을 이끌어 낼 수 있습니다.

1. # 헤더

프롬프트의 뼈대를 형성합니다.

#역할
당신은 10년 경력의 관세사 입니다
#작업
Hs CODE 분류하세요
#출력 형식
json으로 반환

2. "" ''' : 코드 블럭

지시문과 데이터를 격리합니다.

아래 python 코드의 버그를 찾아주세요
'''python
def calculate_tax(price) :
  return price*0.1
'''

3. > : 인용문

핵심 조건이나 맥락을 강조합니다.

>반드시 한국어로만 답변하세요
>추측성 내용은 포함하지 마세요

4. - : 목록

제약 조건과 세부 규칙을 나열합니다.

- 답변은 500자 이내
- 전문 용어 사용 시 괄호 설명
- 예시는 반드시 무역 도메인 선택

5. | | : 표

입출력 예시를 구조화합니다.

| 품목 | Hs Code | 관세율 |
| 티셔츠 | 6109.10 | 13% |
| 바지 | 6111.10 | 15% |

6. 변수와 강조

특정 단어를 굵게 하거나 기울입니다. ( **글자** : 글자 굵게 표시 , *글자* : 기울임)

[분석대상]에 대해 반드시 아래 순서대로 분석하세요

5. 글의 표현 강도와 중요도를 조절하는 방법

지시문의 어미와 부사를 조절하여 AI의 행동 강도를 결정합니다. 강도를 높일수록 AI는 해당 지시를 '절대적 규칙'으로 인식합니다.

1. 강도 단계별 표현

단계 표현 방식 실무 적용 예시
1단계 (권장) 가능하면, 되도록, ~하는 편이 좋습니다 가능하면 한국어로 답변해주세요
2단계 (일반) ~하세요, ~해주세요 한국어로 답변하세요
3단계 (강조) 반드시, 꼭, 항상 반드시 한국어로 답변해주세요
4단계 (절대) 절대로, 어떤 경우에도, 예외 없이 어떤 경우에도 영어를 사용하지 마세요

2. 우선순위 (이모지 활용)

🔴
최우선 (절대 조건)

개인 정보 절대 출력하지 않기, 사실과 다른 내용 생성하지 않기

🟡
중요 (일반 조건)

답변은 500자 이내로 작성, 전문 용어 괄호 설명 추가

🟢
선택 (권장 조건)

가능하면 실제 사례 포함, 표나 목록을 활용하면 더 좋다

6. 톤(말투) 설정 전략

말투 하나로 같은 정보도 전혀 다른 느낌으로 전달됩니다. 상황(유머러스, 따뜻함, 사무적 등)에 맞춰 [처리] 지침을 내리세요.

원본 메시지

"올해 들어 회사는 큰 변화를 맞이했습니다. 이제 새로운 전략과 목표를 설정하고, 모든 직원이 함께 나아가려고 합니다."

다양한 톤 적용 예시

원본 글을 다음의 각 톤에 맞춰 수정하도록 요청할 수 있습니다.

🤣 1. 유머러스한
😢 2. 감정적인
☀️ 3. 따뜻한
😏 4. 아이러니한
🌹 5. 로맨틱한

실제 프롬프트 작성법

#입력: [원본 메시지 내용]
#처리: 위 글을 3. 따뜻한 톤으로 수정해 주세요. 사내 게시판에 올릴 용도입니다.

7. 대안 접근법 패턴 : 모든 해결책 탐색

AI가 임의로 하나의 답을 선택하게 두지 마세요. 모든 대안을 나열하고 비교하게 함으로써 의사결정의 주도권을 확보하는 전략입니다.

일반 프롬프트

"어떻게 하면 좋을까?"
→ AI가 임의로 1개 선택

대안 프롬프트

"모든 대안을 비교해줘"
→ 사람이 최종 선택권 보유

패턴 1

단순 나열형

해결책을 정해진 형식에 맞춰 다각도로 나열하여 비교합니다.

Prompt Structure
  • 방법 {번호}:{이름}
  • 핵심 원리 / 적합한 상황
  • 단점 / 난이도
#문제: 수출 통관 서류 자동 생성
패턴 2

축 기반 비교형

개발 속도, 비용 등 특정 평가 기준을 세워 정밀 분석합니다.

Evaluation Criteria
개발속도비용 유지보수확장성
#대상: FASTAPI, DJANGO 등
패턴 3

상황별 분기형

가상의 독립된 상황들을 제시하고 최적의 도구를 추천받습니다.

Scenario Context
상황 A: 스타트업, 빠른 MVP
상황 B: 10년치 통계 쿼리
#결과: 1순위 / 2순위 추천 도구
패턴 4

역발상 대안형

해결책 외에 "하지 않기"나 "문제 재정의"를 포함합니다.

Critical Thinking

"문제를 해결하는 대신,
문제가 발생하지 않게 재정의"

#문제: 수입 통관 시간 지연

8. 이용자 페르소나 패턴 : 맞춤형 답변 유도

AI의 역할(페르소나) 정의만큼 중요한 것은 답변을 듣는 '대상'을 정의하는 것입니다. 청취자의 수준과 상황을 명시하면 훨씬 더 공감 가고 실용적인 답변을 얻을 수 있습니다.

대상별 맞춤 지시

질문이 같아도 '이용자'가 누구인지에 따라 AI는 용어 선택과 설명의 깊이를 조절합니다.

질문: "블록체인"이 무역 실무에 어떻게 활용되나요?
대학생 무역을 처음 배우는 초보자 대상 설명
수출팀장 중소기업 실무 및 운영 효율 중심 설명
IT결정자 금융 기관 시스템 도입 및 기술적 관점 설명

이용자 정보 (User Profile)

이용자의 상황을 상세히 기술할수록 답변의 만족도가 상승합니다.

  • 직책: 중소기업 대표 (제조업, 직원 30명)
  • 경험: 국내 영업 10년, 해외 수출 초보
  • 우려사항: "복잡하고 어려울 것 같다"는 막연한 두려움
  • 제약조건: 바빠서 긴 글을 읽지 못함
Tip : 전문 용어를 최소화하고 수출 3단계를 각 3줄 이내로 설명하세요.

이용자 페르소나 설계 체크리스트

프롬프트를 작성하기 전, 다음 질문들을 통해 대상을 구체화해 보세요.

기초 정보

직업과 역할은 무엇인가? 주제에 대한 사전 지식은 어느 정도인가?

목표와 행동

이 정보로 무엇을 결정하는가? 현재 얼마나 급한 상태인가?

선호 형식

얼마나 긴 답변을 읽을 수 있는가? 절대 쓰면 안 되는 표현이 있는가?

심리적 장벽

가장 두려워하는 것은 무엇이며, 가장 원하는 결과는 무엇인가?

9. 레시피 패턴 : 목표 달성을 위한 작업 단계 설계

두루뭉술한 답변 대신, 목표 지점까지 한 걸음씩 나아갈 수 있는 구체적인 '순서'와 '방법'을 요구하는 패턴입니다. AI에게 실행 가능한 로드맵을 그려달라고 요청하세요.

일반 질문 (개념 설명)
"수출 자동화 어떻게 해?"

* 답변 예: "API를 활용하거나 파이썬을 배우세요..."

레시피 패턴 (행동 지침)
"수출 자동화를 위한 단계별 절차를 주세요"

* 답변 예: "STEP 1. 관세청 API 연동, STEP 2. 데이터 추출..."

예시 1. 프로그램 제작 레시피

목표 : 환경 구축 및 라이브러리 설치
현재 상태

10. 뒤집힌 상호작용 패턴 : AI가 묻고 사용자가 답한다

사용자가 정보를 다 채우지 못했을 때 유용한 패턴입니다. AI에게 먼저 질문을 던지게 하여 완벽한 정보를 수집한 후 최상의 결과물을 도출하게 합니다.

방법 1. 일괄 질문 (질문 먼저, 답은 나중)

필요한 모든 정보를 한꺼번에 질문 리스트로 받습니다.

Prompt: "제가 원하는 결과물은 {목표}입니다. 바로 만들지 말고, 먼저 필요한 정보를 질문해 주세요. 모든 정보가 수집된 후 결과물을 작성하세요."

방법 2. 순차 질문 (한 번에 하나씩)

채팅하듯 질문을 하나씩 주고받으며 심도 있게 정보를 수집합니다.

Prompt: "제가 원하는 결과물은 {목표}입니다. 한번에 하나의 질문만 하세요. 제가 답하면 다음 질문으로 넘어가고 정보가 충분해지면 결과물을 작성하세요."

실무 적용 예시

예시 1. 일괄 질문 - 수출 제안서 작성

"수출 제안서를 작성해주세요. 바로 작성하지 말고 아래 항목들을 질문해서 제 답변을 모두 수집한 후 작성하세요."
수집 필요 정보: 제품정보, 타깃 바이어, 핵심 강점, 가격 정책, 납기 조건

Expected AI Questions List:
  • AI Q1: 제안할 제품의 주요 스펙과 특징은 무엇인가요?
  • AI Q2: 제안서를 받을 국가 및 바이어 기업의 성격은 어떠한가요?
  • AI Q3: 타사 대비 우리 제품만이 가진 독보적인 강점은 무엇인가요?
  • AI Q4: 희망하는 수출 단가(Incoterms 포함)와 최소 주문 수량은요?
  • AI Q5: 발주 후 선적까지 소요되는 기간 및 물류 조건은 무엇인가요?
  • (일괄 답변 후...) → 제안서 작성 시작
예시 2. 순차 질문 - Python 학습 계획서

"Python 학습 계획서를 만들어주세요. 단, 한 번에 하나의 질문만 하고 내 답변을 들은 뒤 다음 질문으로 넘어가세요. 5~7개의 질문이 끝나면 맞춤형 계획서를 작성해주세요."

Expected Interaction Flow:
  • AI Q1: 현재 파이썬을 어느 정도 알고 계신가요? (완전 초보/기초 문법 숙지 등)
  • AI Q2: 학습의 주요 목적은 무엇인가요? (데이터 분석/웹 개발/업무 자동화 등)
  • AI Q3: 하루에 학습에 투자할 수 있는 시간은 어느 정도인가요?
  • (질문 반복 후...) → 맞춤형 로드맵 생성 시작

11. 인지 검증자 패턴 : 복잡한 문제의 논리적 해체

복잡한 질문을 받았을 때 AI가 즉각 대답하며 발생할 수 있는 오류와 누락을 방지합니다. 문제를 하위 질문으로 분해하여 단계별로 검증한 뒤 통합된 결론을 도출하게 합니다.

일반 패턴 (즉각 대답)

복잡한 질문 입력 시 즉시 답변을 생성하여 논리적 비약이나 누락 가능성이 높음

인지 검증자 패턴 (분해 및 통합)

하위 질문 분해 → 개별 검증 → 최종 답안 통합 과정을 거쳐 정확도 극대화

방법 1. 명시적 분해 요청

AI에게 문제를 쪼개서 생각할 시간을 명시적으로 부여합니다.

"아래 질문에 바로 답하지 말고 먼저 이 질문을 정확히 답하기 위해 필요한 하위 질문 목록을 만들고 각 질문에 순서대로 답한 뒤 최종 통합 답변을 제시하세요."

방법 2. 전문가 검증 방식

전문가적 사고방식을 요구하여 판단 근거와 가정의 명확성을 높입니다.

"{분야} 전문가처럼 이 문제를 다루세요. 확신이 없는 부분은 가정으로 명시하고 판단 근거를 제시하면서 단계적으로 결론에 도달하세요."

실무 적용 시나리오

Case Study: 해외 생산 법인 설립 분석

질문: "우리 회사가 베트남에 생산 법인을 설립하면 세금, 인력, 물류 측면에서 유리한가요?"

Cognitive Verification Steps:
  • Step 1. 하위 질문 도출: 베트남의 법인세 혜택은? 최저임금 및 숙련도 수준은? 주요 항구와의 거리 및 물류 인프라는?
  • Step 2. 개별 검증: 각 하위 질문에 대해 최신 데이터와 규정을 바탕으로 독립적 답변 생성
  • Step 3. 최종 통합: 세금, 인력, 물류 데이터를 결합하여 종합적인 설립 타당성 결론 도출
  • → 결과: 단순 요약을 넘어선 정밀한 전략 보고서 완성
